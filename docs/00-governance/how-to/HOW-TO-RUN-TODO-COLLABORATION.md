# HOW TO RUN TODO COLLABORATION

## 1. Purpose

本文档说明：AI、人类和 `TODO.md` 如何协作完成从宏观任务到执行任务的转换。

目标是让 `TODO.md` 既保持任务队列的清晰，又能承载人机达成共识前的宏观任务入口。

## 2. Scope

当前范围：

- AI 将宏观任务写入 `TODO.md`
- 人类阅读并与 AI 讨论 `TODO.md` 项
- AI 将达成共识的宏观任务拆解为可执行 TODO
- 人类审阅拆解后的 TODO
- AI 执行并在完成后删除 TODO

不在范围内：

- 不替代具体业务需求文档
- 不替代架构红线或文档规则
- 不记录已完成任务历史
- 不把低置信度观察项写入 `TODO.md`

## 3. Bounded Context

`TODO.md` 是未关闭任务面板。

在 AI 协作场景中，`TODO.md` 承载两类未关闭任务：

- 宏观任务：已经值得讨论，但尚未拆解成执行步骤
- 执行任务：已经达成共识，可以进入实现、测试、文档同步和提交

宏观 TODO 不是执行许可。拆解并经人类审阅后的 TODO，才是执行入口。

## 4. When To Use

- AI 通过扫描、讨论或复盘发现一个值得推进的宏观任务
- 人类希望先围绕任务目标、边界和优先级形成共识
- 当前任务需要先拆解，再进入实现
- 当前任务可能影响架构、文档、测试或多个模块

## 5. Do Not Use For

- 已经非常明确的一步式小改动
- 低置信度、暂时无法行动的观察项
- 纯讨论材料、灵感记录或完成历史
- 尚未确认是否属于本仓库范围的想法

## 6. Pre-Checks

写入宏观 TODO 前，AI 固定先判断：

1. 该任务是否已经足够重要，值得打扰人类阅读。
2. 该任务是否能用一句话说明问题或目标。
3. 该任务是否存在明确的下一步讨论点。
4. 该任务是否不是低置信度观察项。
5. 该任务是否不应直接进入执行。

只要以上任一问题不能明确回答，就不要写入 `TODO.md`。

## 7. Steps

### 7.1 Add A Macro TODO

AI 可以将宏观任务写入 `TODO.md`。

宏观 TODO 必须包含：

- 任务目标
- 当前判断
- 需要人类确认的问题

宏观 TODO 不得包含过细执行步骤，不得伪装成已经达成共识的实现计划。

### 7.2 Discuss The Macro TODO

人类阅读宏观 TODO 后，与 AI 讨论以下内容：

- 任务是否值得做
- 边界是否正确
- 优先级是否正确
- 验收口径是否清楚
- 是否需要补充文档、测试或治理规则

讨论完成前，AI 不得把宏观 TODO 当作执行许可。

### 7.3 Split Into Executable TODO

达成共识后，AI 将宏观 TODO 拆解为可执行 TODO。

拆解后的 TODO 必须满足：

- 每项都有明确动作
- 每项可以独立判断完成或未完成
- 每项边界清楚，不混合不相关判断
- 每项能对应到代码、测试、文档或提交动作

### 7.4 Review The Split TODO

人类审阅拆解后的 TODO。

审阅重点：

- 任务粒度是否合适
- 执行顺序是否合理
- 是否遗漏验收条件
- 是否把讨论结论拆偏
- 是否引入了不必要的扩展

审阅通过后，AI 才进入执行。

### 7.5 Execute And Close

AI 按确认后的 TODO 执行。

执行完成后：

- 已完成 TODO 必须删除
- 未完成部分必须拆分或收窄为剩余任务
- 完成历史保留在 commit 或 PR 中
- 删除 TODO 应与对应代码、测试、文档修改放在同一闭环提交中

## 8. Files To Touch

本流程通常涉及：

- 根目录 `TODO.md`
- 与任务相关的业务代码、测试或文档
- 收口时可能涉及的 commit

本流程通常不要求新增治理文档。只有当讨论结论形成稳定规则时，才同步治理文档。

## 9. Common Mistakes

- AI 将宏观 TODO 直接当作执行许可
- AI 把低置信度观察项写入 `TODO.md`
- 人类尚未确认边界，AI 就开始拆解或实现
- 拆解后的 TODO 粒度过细，制造噪音
- 拆解后的 TODO 混入未经讨论的新目标
- 完成后在 `TODO.md` 保留打勾历史

## 10. Verification

进入执行前固定自检：

1. 宏观任务是否已经经过人机讨论。
2. 拆解后的 TODO 是否经过人类审阅。
3. 每个执行 TODO 是否能独立验收。
4. TODO 中是否没有完成历史和低置信度观察项。
5. AI 是否只执行已经确认的 TODO。

## 11. Commit Guidance

如果本流程只修改 `TODO.md`，提交信息应说明任务协作状态变化。

如果本流程完成了代码、测试或文档改动，应按对应任务闭环提交，并在同一提交中删除或收窄已完成 TODO。
